# Quran Hifz Companion

تطبيق ويب عربي RTL لإدارة مشروع حفظ القرآن على المدى الطويل. يركز على الحفظ الجديد، المراجعة، الاختبارات، الوقفات، الصفحات الضعيفة، التخطيط الأسبوعي، والتحليلات ضمن تجربة `local-first` تعمل محليًا بدون Backend في هذه النسخة.

## الهدف من المشروع

بناء أداة شخصية عملية تساعد المستخدم على:

- متابعة الحفظ الجديد والمراجعة بشكل يومي.
- معرفة مواضع الضعف والصفحات المتأخرة بوضوح.
- تنظيم المشروع على المدى الطويل بدل الاعتماد على التتبع الذهني فقط.
- توفير بنية قابلة للتوسع لاحقًا إلى مزامنة سحابية أو لوحة معلم أو Backend فعلي.

## المميزات الرئيسية

- لوحة رئيسية تعرض التقدم الحالي، المؤشرات المهمة، والتنبيهات الذكية.
- إدارة جلسات الحفظ والمراجعة والاختبار والتجويد والوقفات.
- صفحة `مراجعة اليوم` بخطط خفيفة وعادية وموسعة.
- مركز للصفحات الضعيفة والحرجة والمتأخرة.
- تخطيط أسبوعي مرن لتوزيع الحفظ والمراجعة.
- تتبع الاختبارات، الأخطاء، ونسبة التحسن.
- تتبع ملاحظات التجويد وملاحظات المعلم.
- تحليلات ورسوم بيانية للتقدم، المراجعة، الالتزام، والتوقعات.
- دليل استخدام داخلي داخل التطبيق.
- بيانات تجريبية جاهزة وميزة `import/export`.

## هيكل المشروع

```text
app/                  مسارات Next.js والصفحات والـ layouts
components/           مكونات الواجهة العامة
data/                 بيانات تجريبية وبيانات مساعدة
features/             ميزات الصفحات وتدفقات الاستخدام
hooks/                Hooks قابلة لإعادة الاستخدام
lib/
  analytics/          حسابات الإحصائيات وتجهيز بيانات الرسوم
  domain/             منطق المجال مثل review engine والتقارير والتخطيط
  persistence/        طبقة التخزين المحلي وواجهة الاستبدال لاحقًا
types/                الأنواع والنماذج Typed Models
utils/                أدوات مساعدة عامة
public/               ملفات ثابتة
```

## طريقة إعداد ملف .env

لا توجد متغيرات بيئة مطلوبة لتشغيل النسخة الحالية، لأن التطبيق يعمل محليًا بالكامل.

إذا أردت تجهيز المشروع للتوسع لاحقًا، استخدم ملف `.env.example` كقالب:

```env
# أمثلة مستقبلية فقط
NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_public_key
DATABASE_URL=your_database_url
```

ثم أنشئ ملف `.env` محليًا عند الحاجة فقط، ولا تضع فيه أي قيمة حقيقية داخل المستودع.

## ملاحظات الأمان

- لا توجد مفاتيح حقيقية أو أسرار مطلوبة في النسخة الحالية.
- يجب إبقاء أي ملف `.env` محليًا فقط وعدم رفعه إلى GitHub.
- ملف `.env.example` يحتوي placeholders فقط.
- المشروع يستخدم `.gitignore` لاستبعاد:
  - `node_modules`
  - `.next`
  - ملفات البيئة
  - السجلات والملفات المؤقتة
  - ملفات التعليمات المحلية مثل `AGENTS.md` و`CLAUDE.md`
- توجد ملفات بناء محلية كبيرة مثل `.next/` ويجب عدم رفعها.

## المشاكل المعروفة

- لا توجد مزامنة سحابية أو حسابات مستخدمين بعد.
- التخزين الحالي محلي عبر `localStorage` فقط.
- لا توجد اختبارات آلية منفصلة بعد.
- لا توجد طبقة Backend أو قاعدة بيانات فعلية في هذه النسخة.
