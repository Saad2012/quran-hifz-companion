# Quran Hifz Companion

تطبيق ويب عربي RTL لإدارة مشروع حفظ القرآن على المدى الطويل. يركز على الحفظ الجديد، المراجعة، الاختبارات، الوقفات، الصفحات الضعيفة، التخطيط الأسبوعي، والتحليلات ضمن تجربة `local-first` مع تسجيل دخول ومزامنة سحابية عبر Supabase.

## الهدف من المشروع

بناء أداة شخصية عملية تساعد المستخدم على:

- متابعة الحفظ الجديد والمراجعة بشكل يومي.
- معرفة مواضع الضعف والصفحات المتأخرة بوضوح.
- تنظيم المشروع على المدى الطويل بدل الاعتماد على التتبع الذهني فقط.
- توفير بنية قابلة للتوسع لاحقًا إلى مزامنة سحابية أو لوحة معلم أو Backend فعلي.

## المميزات الرئيسية

- لوحة رئيسية تعرض التقدم الحالي، المؤشرات المهمة، والتنبيهات الذكية.
- إدارة جلسات الحفظ والمراجعة والاختبار والتجويد والوقفات.
- صفحة `مراجعة اليوم` بخطط خفيفة وعادية وموسعة.
- مركز للصفحات الضعيفة والحرجة والمتأخرة.
- تخطيط أسبوعي مرن لتوزيع الحفظ والمراجعة.
- تتبع الاختبارات، الأخطاء، ونسبة التحسن.
- تتبع ملاحظات التجويد وملاحظات المعلم.
- تحليلات ورسوم بيانية للتقدم، المراجعة، الالتزام، والتوقعات.
- دليل استخدام داخلي داخل التطبيق.
- بيانات تجريبية جاهزة وميزة `import/export`.
- تسجيل دخول بالبريد الإلكتروني وكلمة المرور.
- تسجيل دخول عبر Google عند تفعيل المزود في Supabase.
- استعادة كلمة المرور وتعيين كلمة جديدة من داخل التطبيق.
- مزامنة سحابية تلقائية مع الاحتفاظ بالنسخة المحلية كخط دفاع أول.
- استعادة المشروع من أي جهاز عند توفر الحساب نفسه.

## طريقة إعداد ملف .env

استخدم ملف `.env.example` كقالب ثم أنشئ `.env.local` محليًا:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_your_public_key_here
```

لا ترفع أي ملف `.env.local` أو `.env` إلى GitHub.

## النشر على Vercel

عند النشر على Vercel أضف متغيرات البيئة التالية إلى المشروع:

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=...
```

ويجب كذلك أن تكون migration الموجودة في:

`supabase/migrations/20260510_auth_and_sync.sql`

مطبقة على مشروع Supabase المستهدف.

ولتفعيل Google Login فعليًا:

- فعّل `Google` داخل `Supabase Auth > Providers`
- أنشئ OAuth Client من نوع `Web application` في Google Cloud
- أضف `Authorized JavaScript origins` لعناوين التطبيق
- وأضف Supabase callback URL الخاص بمشروعك إلى `Authorized redirect URIs`
- ثم ضع `Client ID` و`Client Secret` داخل إعداد مزود Google في Supabase

وإذا أردت أفضل توافق مع رسائل استعادة كلمة المرور في Supabase، اضبط قالب
`Reset Password` ليعبر عبر المسار:

`/auth/confirm`

بحيث يصل المستخدم بعد التحقق إلى:

`/reset-password`

## ملاحظات الأمان

- لا تضع أي مفاتيح أو أسرار حقيقية داخل المستودع.
- يجب إبقاء أي ملف `.env` أو `.env.local` محليًا فقط وعدم رفعه إلى GitHub.
- ملف `.env.example` يحتوي placeholders فقط.
- المشروع يستخدم `.gitignore` لاستبعاد:
  - `node_modules`
  - `.next`
  - ملفات البيئة
  - السجلات والملفات المؤقتة
  - ملفات التعليمات المحلية مثل `AGENTS.md` و`CLAUDE.md`
- ملفات `Serena` مرفوع منها فقط ما يفيد الاستعادة، بينما الكاش والسجلات مستبعدة.
- توجد ملفات بناء محلية كبيرة مثل `.next/` ويجب عدم رفعها.

## المشاكل المعروفة

- لا توجد اختبارات آلية منفصلة بعد.
- المزامنة الحالية تعتمد على لقطة JSON واحدة لكل مستخدم، وليست بعد تفكيكًا كاملًا لكل كيان في جداول مستقلة.
- لا توجد بوابة معلم أو صلاحيات متعددة المستخدمين بعد.
- نجاح استعادة كلمة المرور يعتمد أيضًا على إعداد قالب البريد في مشروع Supabase إذا أردت تدفق `token_hash` الموصى به.
